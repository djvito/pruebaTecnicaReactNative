# React Native Frontend Technical Test

## Introduction
This project is a comprehensive technical test for frontend development using React Native. It emphasizes adhering to best coding practices, crafting UIs without relying on pre-built style frameworks, proficient exception handling, and achieving at least 70% test coverage in unit testing.

## Project Setup
1. **Clone the Repository**: 
   `git clone https://github.com/djvito/pruebaTecnicaReactNative`.
2. **Install Dependencies**:
   - Delete the `node_modules` folder if it exists.
   - Run `npm install` or `yarn install` to reinstall dependencies.

## Environment Configuration
- Ensure you follow the React Native - Environment Setup guide up to "Creating a new application" before proceeding.

## Running the Application
- **Start the Metro Server**:
   - Use `npm start` or `yarn start`.
- **Launch the App**:
   - For Android: `npm run android` or `yarn android`.
   - For iOS: `npm run ios` or `yarn ios`.
- Note: The app should appear in your Android Emulator or iOS Simulator.

## Unit Testing and Coverage
- Execute tests using `npm test` or `yarn test`.
- Aim for a minimum of 70% code coverage. Use coverage tools to track this metric.

## Code Modification
- **File to Edit**: Make changes in `App.tsx`.
- **Viewing Changes**:
   - Android: Double press 'R' or use "Reload" in the Developer Menu.
   - iOS: Press Cmd + R in the iOS Simulator.

## Troubleshooting
- For common issues, refer to the Troubleshooting section.
- Ensure all environment setups are correctly configured.

# Prueba Técnica de Frontend con React Native

## Introducción
Este proyecto es una prueba técnica exhaustiva para el desarrollo de frontend con React Native. Se centra en seguir las mejores prácticas de codificación, desarrollar interfaces de usuario sin depender de frameworks de estilos predefinidos, manejar excepciones eficientemente y lograr al menos un 70% de cobertura en pruebas unitarias.

## Configuración del Proyecto
1. **Clonar el Repositorio**: 
   `git clone https://github.com/djvito/pruebaTecnicaReactNative`.
2. **Instalar Dependencias**:
   - Elimina la carpeta `node_modules` si existe.
   - Ejecuta `npm install` o `yarn install` para reinstalar las dependencias.

## Configuración del Entorno
- Asegúrate de seguir la guía de Configuración del Entorno de React Native hasta "Crear una nueva aplicación".

## Ejecución de la Aplicación
- **Iniciar el Servidor Metro**:
   - Usa `npm start` o `yarn start`.
- **Lanzar la App**:
   - Para Android: `npm run android` o `yarn android`.
   - Para iOS: `npm run ios` o `yarn ios`.
- Nota: La app debería aparecer en tu Emulador de Android o Simulador de iOS.

## Pruebas Unitarias y Cobertura
- Ejecuta las pruebas con `npm test` o `yarn test`.
- Apunta a un mínimo del 70% de cobertura de código. Utiliza herramientas de cobertura para seguir esta métrica.

## Modificación del Código
- **Archivo para Editar**: Realiza cambios en `App.tsx`.
- **Visualizar Cambios**:
   - Android: Presiona 'R' dos veces o usa "Recargar" en el Menú de Desarrollador.
   - iOS: Presiona Cmd + R en el Simulador de iOS.

## Solución de Problemas
- Para problemas comunes, consulta la sección de Solución de Problemas.
- Asegúrate de que todos los aspectos del entorno estén configurados correctamente.
